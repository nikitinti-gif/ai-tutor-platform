# DOMAIN MODEL

Версия: 1.0

Этот документ определяет язык платформы.

Если термин отсутствует здесь — его нельзя использовать в коде.

Все разработчики, AI-агенты и преподаватели используют одинаковые определения.

---

# Student

Ученик.

Человек, проходящий индивидуальную образовательную траекторию.

Student является владельцем собственного Learning DNA.

---

# Teacher

Преподаватель.

Эксперт, который принимает окончательные педагогические решения.

Teacher может:

• подтверждать рекомендации AI

• изменять рекомендации AI

• отклонять рекомендации AI

Каждое действие Teacher становится новым источником знаний для системы.

---

# Parent

Родитель.

Наблюдатель образовательного процесса.

Parent получает отчёты, аналитику и прогнозы.

Parent никогда не изменяет Learning DNA.

---

# Learning DNA

Цифровая модель текущего состояния обучения ученика.

Learning DNA содержит:

• знания

• навыки

• учебные сигналы

• историю

• мотивацию

• прогноз

Learning DNA является единственным источником истины о состоянии ученика.

---

# Learning Evolution

История изменений Learning DNA.

Позволяет ответить:

Как изменился ученик?

Почему он изменился?

Что повлияло на изменение?

---

# Topic

Тема экзамена.

Например:

• Проценты

• Дроби

• Линейные уравнения

• Квадратные уравнения

Topic — это раздел программы.

---

# Skill

Навык.

Минимальная единица обучения.

Пример.

Тема:

Квадратные уравнения

состоит из навыков

• вычисление дискриминанта

• определение числа корней

• проверка корней

• оформление решения

Navigator принимает решения именно по навыкам.

Не по темам.

---

# Learning Mission

Учебная миссия.

Набор действий,

которые ученик должен выполнить

для развития конкретного навыка.

Learning Mission включает:

• цель

• теорию

• упражнения

• проверку

• критерии успешности

---

# Mission Result

Результат выполнения миссии.

Mission Result отвечает:

Удалось ли развить навык?

---

# Observation

Наблюдение.

Объективный факт.

Observation никогда не содержит выводов.

Пример.

"Ученик три раза подряд ошибся."

---

# Evidence

Доказательство.

Интерпретация одного или нескольких Observation.

Evidence объясняет,

почему Observation является значимым.

---

# Reasoning

Педагогическое рассуждение.

Объяснение,

каким образом опытный преподаватель интерпретирует Evidence.

Reasoning всегда должно быть понятно человеку.

---

# Decision

Педагогическое решение.

Ответ на вопрос:

Что делать дальше?

Decision никогда не принимается без Observation,

Evidence

и Reasoning.

---

# Action Plan

План действий.

Конкретный список действий,

которые необходимо выполнить

после Decision.

---

# Learning Signal

Учебный сигнал.

Любое событие,

которое изменяет понимание системой состояния ученика.

Learning Signal заменяет понятие "ошибка".

---

# Knowledge Graph

Граф взаимосвязей между темами,

навыками

и зависимостями.

Knowledge Graph помогает строить траекторию обучения.

---

# Learning Trajectory

Последовательность освоения навыков.

Trajectory отвечает

в каком порядке

ученик будет изучать материал.

---

# Learning Navigator

Механизм,

который выбирает

следующее наиболее полезное действие.

Navigator не использует интуицию.

Он использует:

Learning DNA

+

Policies

+

Evidence

+

Reasoning.

---

# Pedagogical Policy

Формализованная педагогическая гипотеза.

Каждая Policy содержит:

Observation

↓

Evidence

↓

Reasoning

↓

Decision

↓

Expected Outcome

Policy можно измерить.

Policy можно улучшить.

---

# Expected Outcome

Ожидаемый педагогический эффект.

Например.

Снизить количество ошибок

по дискриминанту

на 50%

за две недели.

---

# Policy Evaluation

Оценка эффективности Policy.

Отвечает:

Сработала ли выбранная стратегия?

---

# Pedagogical Decision Engine

Ядро платформы.

Преобразует

Observation

↓

Evidence

↓

Reasoning

↓

Decision

↓

Action Plan

---

# AI Engine

Модуль взаимодействия с LLM.

AI Engine:

• анализирует решения

• генерирует задания

• составляет отчёты

AI Engine не принимает педагогических решений.

---

# Learning Operating System

Главное ядро платформы.

Learning OS объединяет:

Learning DNA

Navigator

Policy Engine

Analytics

Mission Engine

Teacher Assistant

---

# Teacher Assistant

AI-помощник преподавателя.

Не заменяет преподавателя.

Помогает:

• проверять

• анализировать

• рекомендовать

• планировать

---

# Parent Report

Объяснимый отчёт для родителей.

Каждый отчёт должен отвечать:

Что сделал ребёнок?

Что изменилось?

Почему это важно?

Что будет дальше?

За что родители платят?

---

# Event

Любое изменение состояния системы.

Каждый Event должен увеличивать понимание ученика.

Если Event ничего нового не сообщает,

он не нужен.

---

# KPI

Измеримый показатель эффективности обучения.

Например.

• рост навыка

• снижение ошибок

• регулярность занятий

• прогноз ЕГЭ

• вовлечённость

---

# Learning Intelligence Platform

Платформа,

которая помогает преподавателю принимать педагогические решения

на основе данных,

аналитики,

AI

и накопленного педагогического опыта.
