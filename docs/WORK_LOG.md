### 2026-07-13 — прокси изолирован для запросов Gemini

- Цель: обойти региональное ограничение Gemini, не направляя Telegram-трафик и остальные соединения бота через прокси.
- Причина прежней ошибки: Google Gen AI SDK наследовал системную переменную прокси со схемой `socks4://`, которую транспорт `httpx` не поддерживает.
- Что изменено: добавлена отдельная настройка `GEMINI_PROXY_URL`; системные proxy-переменные для Gemini теперь игнорируются; разрешены `http://`, `https://` и `socks5://`; для SOCKS5 добавлена зависимость `httpx[socks]`; некорректная схема или адрес дают понятную конфигурационную ошибку.
- Затронутые файлы: `src/ai_engine/llm_client.py`, `requirements.txt`, `docs/WORK_LOG.md`.
- Проверка: Python-синтаксис проверен локально; реальный запрос через пользовательский прокси ещё не выполнен.
- Статус: требует проверки.
- Следующий шаг: установить зависимости, указать `GEMINI_PROXY_URL` в локальном `.env` и повторить `python tests/run_gemini_evaluation.py`.

### 2026-07-13 — HTTP-прокси не прошёл TLS-рукопожатие

- Цель: выполнить первый из 15 синтетических запросов Gemini через локальный прокси.
- Фактический результат: схема `http://127.0.0.1:10808` была принята клиентом, но запрос остановился на TLS-рукопожатии внутри `httpcore._sync.http_proxy` и был прерван пользователем через `Ctrl+C`.
- Диагноз: порт доступен, но, вероятнее всего, не является HTTP CONNECT-прокси; требуется проверить его как SOCKS5-порт и отдельно подтвердить выходной IP.
- Статус: требует проверки.
- Следующий шаг: проверить порт командой `curl` через SOCKS5, затем установить `GEMINI_PROXY_URL=socks5://127.0.0.1:10808` и повторить оценку.
